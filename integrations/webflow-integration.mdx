## Overview

The Webflow CMS integration lets you publish SEO-optimized content directly to your Webflow site's CMS collection — as a draft or live — straight from Pierview. Set up your connection once, then export any generated article with a single click, or let your content schedule publish automatically.

## Step 1: Generate a Webflow Site API Token

1. In Webflow, open your site and go to **Site settings → Apps & integrations → API access**.
2. Click **Generate API token**, name it `Pierview`, and grant these scopes:
   - **CMS** — Read and write (`cms:read`, `cms:write`)
   - **Sites** — Read (`sites:read`)
3. Copy the token.

<Warning>
  Keep the token dialog open until you've pasted it into Pierview — Webflow only shows the token once.
</Warning>

## Step 2: Connect Pierview to Webflow

1. In Pierview, go to [Settings → Integrations → CMS Integrations](https://pierview.ai/dashboard/configuration/integrations) and click **Connect** next to Webflow.
2. **Paste your Site API token**, then click **Fetch Sites** and pick your site.
3. Select the **CMS collection** where content should be published (e.g. "Blog Posts").
4. Map the fields:
   - **Title → `Name`** (the collection's name field)
   - **Slug → `Slug`**
   - **Content → a Plain-Text field** (see the next step — this is important for tables)

## Step 3: Set up the body field so tables render (one-time)

Webflow's **Rich Text** field does **not** support tables — it strips them out. To publish clean content that includes tables, Pierview writes the article as HTML into a **Plain-Text field**, which you render with a **Custom Code Embed** in your blog template. This is a one-time setup in the Webflow Designer:

1. In your collection, add a new **Plain Text** field — name it e.g. **"Article HTML"**, and set its text type to **Multiple line (long text)**, not single line. Map Pierview's **Content** field to this in Step 2.

   <Warning>
     The field **must** be multi-line. A single-line Plain Text field will save drafts fine but **fail on live publish** with "Expected value to be a single line," because the article HTML contains line breaks.
   </Warning>
2. Open your **blog post template page** in the Designer.
3. Drag in an **Embed** element (Add panel → Components → Embed / Custom Code).
4. In the embed, **bind it to your "Article HTML" field** (click the purple "+ Add field" / data-binding control and choose the field).
5. Publish your site.

<Tip>
  Pierview wraps the article in a `.pv-content` container and includes basic table styling, so tables look clean out of the box.
</Tip>

<Note>
  **Already have a blog with existing posts?** Your existing posts likely use the collection's **Rich Text** field (e.g. `Post Body`). Pierview writes to the **new** `Article HTML` field instead and never touches your old field, so existing content is safe. To make one template page render both old and new posts cleanly, add a **conditional visibility** rule to each element (select the element → **Settings panel → Conditional Visibility**):

  - **Rich Text element** → show only **when `Post Body` is set**
  - **Code Embed element** → show only **when `Article HTML` is set**

  This way old posts render via Rich Text, Pierview posts render via the Embed, and neither shows an empty container. (If you skip this, both still work — you may just see a small empty gap where the unused element sits.)
</Note>

## Step 4: Match your brand's table styles (optional but recommended)

Pierview ships sensible default table styles, but you'll likely want tables to match your site's colors and fonts. Because all Pierview content is wrapped in a `.pv-content` class, you can override the defaults from a single place.

**In Webflow Designer → Site Settings → Custom Code → Head Code**, add:

```css
<style>
/* ── Pierview content styles ── */
.pv-content table {
  border-collapse: collapse;
  width: 100%;
  margin: 1rem 0;
}
.pv-content th,
.pv-content td {
  border: 1px solid #e2e2e2; /* ← replace with your border color */
  padding: 8px 12px;
  text-align: left;
}
.pv-content th {
  background: #f7f7f7;        /* ← replace with your header background */
  font-weight: 600;
}
</style>
```

Swap `#e2e2e2` and `#f7f7f7` for your brand's colors. Everything else — headings, paragraphs, bold text, lists — inherits your site's existing typography automatically, so it will already match without any extra work.

<Note>
  **Editing workflow:** Pierview is the editor for this integration. Make changes in Pierview and re-publish — the Webflow item updates automatically. Because the article body lives inside a Code Embed, it is not directly editable in the Webflow Designer, which is by design.
</Note>

## You're Connected!

Open any content generation and click **"Publish"** in the top-right sidebar. Choose **Save as Draft** (staged in Webflow, not live) or **Publish Live**. Re-publishing the same article updates the existing item instead of creating a duplicate. Scheduled content publishes automatically once the connection is enabled.

## Troubleshooting

<AccordionGroup>
  <Accordion title="'Publish your Webflow site once' error when publishing live">
    Webflow can only create **live** items on a site that has been published at least once. Publish your site in Webflow (the **Publish** button), then try **Publish Live** again. Saving as a **draft** always works, even before the first publish.
  </Accordion>
  <Accordion title="Tables aren't showing / content looks like raw HTML">
    Make sure you mapped **Content** to a **Plain-Text** field (not Rich Text) and that your blog template has a **Custom Code Embed** bound to that field (Step 3). Rich Text fields strip tables, which is why the Embed approach is required.
  </Accordion>
  <Accordion title="Articles not appearing">
    - Verify your API token is correct and has `cms:read`, `cms:write`, and `sites:read` scopes
    - Confirm the correct site and collection are selected
    - Check that the integration is enabled (toggle is on)
    - For live publishing, ensure the site has been published at least once
  </Accordion>
  <Accordion title="Duplicate slugs">
    Webflow automatically appends a unique suffix to a duplicate slug — no action needed.
  </Accordion>
  <Accordion title="Tables look different from the rest of my site">
    Pierview's default table styles use neutral gray colors that may not match your brand. Follow Step 4 to add your own `.pv-content` CSS overrides in Webflow's Custom Code settings — this lets you control border colors, header backgrounds, fonts, and spacing to match your design system exactly.
  </Accordion>
  <Accordion title="I can't edit the article body in Webflow Designer">
    This is expected. Because the article body is published as HTML into a Code Embed, it is not directly editable in Webflow. All editing happens in Pierview — make your changes there and click **Re-save** to push the update to Webflow.
  </Accordion>
</AccordionGroup>
